### 1. Аннотация
Данный программный комплекс предназначен для автоматизации процессов учета, обработки и визуализации статистических данных футбольного чемпионата, состоящего из 20 команд и 380 матчей (полный игровой цикл). Система обеспечивает расчет турнирных показателей в соответствии с регламентом FIFA/UEFA, поддерживает ведение истории по турам и предоставляет инструменты аналитики.

### 2. Системные требования

#### 2.1. Программные требования
*   Операционная система: Microsoft Windows 10 версии 1809 или выше, Windows 11.
*   Среда выполнения: .NET Runtime 8.0.
*   Среда разработки (для сборки): Microsoft Visual Studio 2022 (версия 17.8 и выше) или JetBrains Rider.
*   Инструментарий сборки: .NET SDK 8.0.

#### 2.2. Аппаратные требования
*   Процессор: x64-совместимый, с тактовой частотой не менее 2.0 ГГц.
*   Оперативная память: не менее 4 ГБ.
*   Свободное дисковое пространство: 200 МБ (включая зависимости).

### 3. Функциональные характеристики

Программное решение реализует следующие алгоритмические задачи:
1.  **Расчет турнирных показателей:** Вычисление очков (Победа — 3, Ничья — 1, Поражение — 0), разницы забитых и пропущенных мячей.
2.  **Многокритериальная сортировка:** Ранжирование команд по убыванию очков, затем по разнице мячей и количеству забитых голов.
3.  **Анализ динамики:** Хранение и извлечение статистических срезов по каждому из 38 игровых туров для построения графиков позиций.
4.  **Поиск и фильтрация:** Выборка данных по заданным атрибутам (название команды, показатели результативности).
5.  **Визуализация:** Построение линейных графиков изменения положения команд в таблице с использованием библиотеки OxyPlot.
6.  **Сериализация:** Сохранение и загрузка состояния системы в формате JSON с обеспечением целостности данных.

### 4. Архитектура и структура проекта

Проект реализован в соответствии с архитектурным паттерном MVVM (Model-View-ViewModel):

*   **Models:** Содержит определения сущностей `Team`, `Match`, `TeamRoundStats` и перечисления `Enums`.
*   **Services:** Реализует бизнес-логику в классе `LeagueManager` и механизмы ввода-вывода в `DataLoader`.
*   **Views:** Описывает пользовательский интерфейс на языке разметки XAML.
*   **Helpers/Utils:** Включает вспомогательные методы расширения, конвертеры типов для UI и константы конфигурации.
*   **Data:** Содержит файл базы данных `data.json` и скрипты генерации тестовых наборов данных.

### 5. Инструкция по установке и развертыванию

1.  **Клонирование репозитория:**
    Использовать систему контроля версий Git для получения исходного кода:
    `git clone [URL_репозитория]`
2.  **Восстановление зависимостей:**
    Перейти в директорию проекта и выполнить команду в терминале:
    `dotnet restore`
3.  **Сборка проекта:**
    Выполнить компиляцию исходного кода:
    `dotnet build --configuration Release`
4.  **Запуск приложения:**
    Запустить исполняемый файл из директории `bin/Release/net8.0-windows/FootballLeague.exe` или через IDE.

### 6. Тестирование системы

Проверка корректности функционирования системы осуществляется следующими методами:

1.  **Модульное тестирование (Unit Testing):** Проверка алгоритмов расчета очков и логики сортировки в классе `LeagueManager`.
2.  **Валидация данных:** При загрузке JSON-файла система проверяет корректность идентификаторов команд и числовых диапазонов забитых мячей.
3.  **Стресс-тестирование:** Проверка производительности при обработке полной базы данных из 380 матчей (время обработки не должно превышать 100 мс на стандартном оборудовании).
4.  **Тестирование пользовательского интерфейса:** Проверка корректности отображения графиков и обновления таблицы при изменении входных данных.

Для запуска автоматизированных тестов (при их наличии в проекте) используетс!я команда:
`dotnet test`

### 7. Обработка исключительных ситуаций

Система включает механизмы перехвата исключений в следующих узлах:
*   Чтение/запись файлов (проверка прав доступа и целостности файла).
*   Парсинг данных (обработка некорректных форматов в JSON).
*   Инициализация графических компонентов (предотвращение сбоев при отсутствии данных для рендеринга).
